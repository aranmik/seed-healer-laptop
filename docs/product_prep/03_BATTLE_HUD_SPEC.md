## 1. 설계 원칙 5

1. **위→아래 = 원인→상태→행동.** 위험의 원인(보스)은 위, 파티 상태는 가운데, 내 행동(스킬·자원)은 엄지 존.
2. **읽기 3순위.** 시선 우선순위는 ①보스 시전바·예고 ②레이드 프레임 ③슬롯 상태. 연출은 이 순위를 침범하지 않는다.
3. **삼중 코딩.** 모든 위험 신호는 색 + 모양 + 아이콘을 함께 사용 (색약 대응 및 소형 화면 식별성).
4. **엄지 존 고정.** 6슬롯·자원·시전 취소는 하단 30%에. 전투 중 상단 터치 요구 최소화 (프레임 탭 제외).
5. **이유는 문장으로.** 거부(쿨다운·마나 부족)와 아군 실수(차단 실패 사유)는 반드시 이유를 표시한다. (R&D 검증: 거부 사유 토스트)

## 3. A · 보스 HUD

- 보스 이름 + 난이도 뱃지(게시판과 동일 아이콘 — 학습 연결).
- HP바: 퍼센트 + 25% 눈금. 위상(페이즈) 전환점이 있으면 눈금에 마름모 표시.
- **광폭화 타이머**: 우측 상시 표시. 15초 미만부터 색 전환 + 테두리 펄스. (전조 #8)
- **다음 패턴 예고 칩** (R&D 승계): "다음: 🔨 강타" — 도감 해금 전에는 "다음: ???"로 표기해 트라이 가치 보존 (Lock 09).
- 일시정지 버튼 (시안 채택).

## 4. B · 보스 시전바

| 종류 | 표현 | 의미 |
|---|---|---|
| 차단 가능 시전 | **파란 바 + ⚡ + 점선 테두리** + 남은 초 | 도적의 차단 대상. 도적 차단CD 칩과 연동 |
| 차단 불가 시전 | **보라 바 + 🚫** + 남은 초 | "막는" 대응이 아니라 "받아낼" 준비 신호 |

- 차단 성공 시: 바가 깨지는 파열 연출 + 성공 SFX + 로그 한 줄.
- 차단 실패 시: 실패 사유를 문장으로 (예: "차단 실패 — 도적이 빙결 상태"). **실수의 이유가 보여야 개입이 성립한다.** (R&D EP05 J3)

## 5. C · 전장

- 보스 중앙, 아군 3 + 사제(후열)가 하단에. 자동 전투 연출(이동·타격·피격)은 가볍게 — 판단 정보는 전부 HUD가 담당.
- **전조 텔레그래프** (전장 안 표시):
  - 광역: 바닥 붉은 링 + ⚠️ + 카운트다운
  - 표식: 대상 머리 위 🔻
  - 폭탄: 대상 위 💣 (프레임 칩과 동시)
  - 강타: 보스 무기 치켜들기 포즈 + 대상 발밑 표식
- 이미지 교체 슬롯: 보스/캐릭터 스프라이트는 플레이스홀더(이모지/도형) → 루미 에셋으로 교체 가능한 구조 (07 문서).

## 6. D · 레이드 프레임 — 이 게임의 심장

**4칸 고정, 1번 칸 = YOU(사제).** "나는 전장 안의 사제다"(Lock 02)를 UI 구조로 번역한 것. R&D의 "아군 4열 + 사제 별도 조작부"에서, Product는 **사제가 프레임 안으로 들어온다.**

### 프레임 해부도 (1칸)

```
┌────────────────────────────┐
│ [초상] 이름          ⚔️ 🎯 │  ← ⚔️=현재 어그로 대상, 🎯=강타 예고 대상
│ HP ▓▓▓▓▓▓▓░░░  8120        │  ← 보호막은 HP바 위 밝은 겹층(잔량 비례)
│ 🌿12s  💀8s  💣3s  ⏳CD14s │  ← 칩 열: HoT · 디버프(남은 초) · 폭탄 · 차단CD
└────────────────────────────┘
```

- **YOU 프레임**: 테두리 금색 등으로 구분 + 마나 미니바 표시 (자기 생존 = 사제의 일, Lock 02).
- **탭 = 대상 지정**: 지정 프레임 발광. P1은 "프레임 선탭 → 스킬 탭" 방식. (스마트 캐스트는 튜닝 후보 — 07 문서 판단 항목)
- **위험 펄스**: HP 30% 미만 시 프레임 테두리 펄스 + 심박 SFX 소량. (R&D 승계)
- **플로팅 숫자**: 피해=적색, 회복=녹색, 흡수=하늘색. 프레임 로컬 풀 + 개수 캡 (R&D 승계 — 성능·가독 보호).
- **칩 규칙**: 동시 표시 최대 3개 + `+N`. 우선순위: 폭탄 > 낙인/저주 > 차단CD > HoT.
- **사망 표현**: 회색 처리 + 🪦. **부활 UI 없음** (Lock 04: 죽기 전에 살리는 게임).

## 7. E · 6스킬 슬롯

시안의 1×6 배열 채택 (Quick Heal / Guardian Shield / Cleanse / Salvation / Blessing / Ring of Light — 초반 6종과 1:1 일치 확인).

### 슬롯 상태 7종

| 상태 | 표현 |
|---|---|
| 준비 완료 | 풀컬러 + 은은한 대기 발광 |
| GCD 중 | 짧은 스윕 오버레이 (R&D: GCD 1.0s 기준) |
| 쿨다운 중 | 어둡게 + 남은 초 숫자 + 시계 스윕 |
| 마나 부족 | 회색 + 파란 테두리 (원인이 마나임을 색으로) |
| 시전 중 | 해당 슬롯 테두리 진행 링 |
| 대상 필요 | 반투명 + 탭 시 "대상을 선택하세요" 토스트 |
| 잠김(튜토리얼) | 자물쇠 오버레이 |

- **거부 사유 토스트** (R&D 승계): 탭했는데 안 나가면 반드시 이유 1줄 ("재사용 대기 3초", "마나 부족").
- 터치 타깃 최소 56px. 슬롯 간격은 오탭 방지 8px 이상.

## 8. F · 자원 & 사제 시전바

- **마나 바**: 수치 병기(72/100 스타일 — 시안 채택). 자연 회복은 바 끝 미세 파동으로 암시.
  - 명칭 노트: 시안의 "faith" 표기는 성장축 '사제 신앙'(Lock 08)과 용어 충돌 → **자원명은 '마나' 유지 권고** (판단 필요 항목).
- **사제 시전바**: 슬롯 위에 표시, **바 탭 = 시전 취소** (R&D 검증). 시전형 스킬의 마나는 **완료 시 소비** — 취소·무산은 시간 손실만 (R&D 설계 결정 승계, 이중 페널티 방지).

## 11. 로그 · 힌트

- **색상 로그**: 최근 3줄, 접기 가능. 피해/회복/차단/정화/사망을 색으로 구분 (R&D 승계).
- **위기 힌트 1줄**: "도적이 얼었다 — 정화하면 차단할 수 있다!" — **초반 30분 커리큘럼 구간 한정.** 이후에는 태그·도감이 그 역할을 맡는다 (Lock 09 정답 공개 금지와의 균형).
