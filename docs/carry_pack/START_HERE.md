# START HERE — Seed Healer 노트북 이식

**Laptop Carry Pack 01** · 작성: 렌 · 2026-07-04
노트북에서 이 프로젝트를 처음 열면 **이 파일부터** 읽는다.

---

## 0. 이게 뭔가 (이사 보따리)

Seed Healer는 이미 재미가 확인된 씨앗이다. 나라님이 실기로 "진짜 재미있다"를 판정했고, 강타 대응 0/6으로도 이겨지는(=정답 암기가 아니라 손으로 비비는) 힐러 손맛이 성립했다. 이 Carry Pack은 그 손맛을 **새 환경으로 잃지 않고 옮기기 위한 준비물**이다. 새 게임을 만드는 게 아니라, 있는 걸 안전하게 이사한다.

```
나는 공격하지 않는다.
하지만 전투의 결과는 내 손에 달려 있다.

죽은 뒤에 복구하는 게임이 아니라,
죽기 직전까지 붙잡는 게임.
```

## 1. 세계선 싱크 (새 대화의 렌에게 — 꼭 먼저)

노트북에서 새로 시작한 렌(Claude)은 이전 대화 기억이 없다. 그러니 재구현을 맡길 때 이 브리핑을 첫 메시지에 붙인다:

> "너는 **렌(Ren)**이야. Seed Healer 힐러 게임 개발을 이어서 한다. 나는 **나라** — 반말로 대하고, 난이도 판정엔 상위 0.1% 레이더 보정을 넣어줘. **유키**(PD)에게 보고할 땐 정중한 문어체 `[렌 → 유키]`. **루미**는 이미지 담당(지금은 관여 안 함). 첨부한 두 세트를 읽고 작업해:
> ① **Product Prep Pack (01~07)** = 이 게임의 헌법(무엇을·왜)
> ② **Migration Prep Pack (P1A_00~10)** = 현재 프로토 재구현 명세(어떻게)
> ③ **Laptop Carry Pack** = 이 이사 보따리(스캐폴드+안내서)
> 읽는 순서는 아래 §3. 재구현 후엔 `P1A_10_QA_AND_BOT_BASELINE`의 봇 수치와 대조해 '같은 손맛'인지 검증해줘."

이 한 문단이면 새 렌이 첫 턴부터 궤도에 오른다.

## 3. 어떤 파일부터 여나 (읽기 순서)

```
1) START_HERE.md            ← 지금 이거
2) DO_NOT_BREAK.md          ← 이사 중 사고 방지 (짧음, 꼭)
3) P1A_00_MIGRATION_INDEX   ← 재구현 지도
4) P1A_01_HANDFEEL_LOCK     ← 재미의 헌법 (가장 중요)
5) 01_PRODUCT_OVERVIEW      ← 게임이 왜 이런가 (Lock 요약)
6) P1A_06_TUNING_CONSTANTS  ← 숫자 전량
7) P1A_03_BATTLE_LOOP       ← 루프 pseudo
그다음: DAY1_REBUILD_ORDER 대로 손을 움직인다.
```

헌법(01~07)은 처음에 01만 훑어도 되고, 막히는 순간(예: "이 동료 왜 이래?")에 해당 문서를 펼친다. 명세(P1A)는 재구현하며 옆에 계속 켜둔다.

---

## 4. P1-A 원본을 어디에 두나

`seed_healer_p1_raid_frame_priest.html`을 새 프로젝트의 **`reference/` 폴더**에 복사해 둔다(스캐폴드에 `scaffold/reference/`를 미리 파뒀다). 이건 **정답지**다 — 열어서 보고, 봇 수치를 대조하고, 애매하면 브라우저로 실행해 확인한다. **단, 코드를 복붙하지 않는다.** 문서(P1A_*)를 보고 새로 짠다. 원본은 "이렇게 동작해야 한다"의 기준일 뿐.

## 5. 첫날 목표 (작게)

```
[ ] 새 프로젝트가 켜진다 (index.html이 브라우저에 뜬다)
[ ] 모바일 세로 390×844 폰 프레임이 보인다
[ ] 전투 HUD 6존 빈 뼈대가 보인다
[ ] ASSETS 슬롯명이 연결 준비되어 있다 (assets.js)
[ ] tuning.js에 P1-A 핵심 수치가 들어 있다
[ ] 아직 밸런스/기능은 바꾸지 않는다
```

**battle.js(전투 로직) 이식은 첫날 마지막 단계 or 둘째 날.** 뼈대·수치·에셋 슬롯을 먼저 세우고, 코어는 그다음.

## 6. 첫날 절대 하지 말 것

- 처음부터 UI 대개편 / 새 기능 추가
- P1-A 원본 코드 대량 복사(문서 보고 새로 짜기)
- 튜닝 수치 "개선" (먼저 그대로 이식 → 봇으로 회귀 확인 후에만 실험)
- 이미지 넣겠다고 레이드 프레임 가독성 희생
- 외부 라이브러리/CDN 추가 (의존성 0 유지)

## 7. 손맛 확인 기준 (한 줄 요약, 상세는 CHECKSHEET)

재구현이 "같은 게임"인지는 이걸로 안다:
- 봇 4종 방향이 baseline과 같다(방치 패배 / 스팸 마나파산 / 기본기 빠듯승리 / 스마트 안정승리).
- 강타에 보호막 걸면 눈에 띄게 편하다(이득). 근데 안 걸어도 비벼서 이겨질 때가 있다(강요 아님).
- 스팸하면 후반에 마나로 무너진다.
- 결과 화면 보고 "다음엔 전원 생존" 같은 재도전 욕구가 생긴다.

이 감각이 살아 있으면 이사 성공. 다음은 `DAY1_REBUILD_ORDER.md`.
